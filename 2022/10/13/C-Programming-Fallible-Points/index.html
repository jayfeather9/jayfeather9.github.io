程设常见易错点总结分析

Posted on 2022-10-13 Edited on 2022-10-15

本文将从基本到复杂（并不严格的划分，看我心情）逐步列举一些大家程设学习中常见的语言易错点，供大家自查。

每一个易错点都会附一份错误代码，并说明代码问题，如果可以的话，最好先自己想想代码出了什么问题，再看说明。

第一章 基本中的基本

在这个条目中列举的，都是一些“只要打出这段语句就应该注意不犯”的错误，同时也是最容易犯的一些错误。

1.5-数据类型的常见问题

1.5-错误代码

题目：输入 $x,y,z$ 三个整数，代表三个人的分数，请计算他们的平均分（保留一位小数），。

1.5-错误点

错误点在于会输出整除值。在C语言中，进行加减乘除等基本运算（用运算符定义的两目运算）时，会按照运算符两侧的数据类型决定计算结果的数据类型。最容易出现的错误就是如题中int除以int，则C语言会输出一个int，表现为整除。具体的决定数据类型的方式为：如果两侧类型相同，则输出类型也为该类型，如果类型不同，则将范围较小的转为范围较大的数据类型后再行计算，返回较大范围的数据。常用存数类型的范围大小关系如下：

$short < int \leq long < long long < float < double$

具体各个数据的存储范围和精度详见课件，long类型不必深究，一般不需要考虑，无符号类型不要和有符号类型混合计算，计算前先统一各个数据的有无符号性。

附：数据类型尽量不要混用、乱用！比如pow函数、sqrt函数等等math.h库的函数返回值类型都是double，尽量不要在需要用int类型的时候使用这些函数！！不然容易因为double的精度问题产生意料不到的错误！！

其他基本要点

  • 注意%运算只能在整数之间进行，否则会CE
  • if里一定要用==，不要用成=
  • scanf里不能像“%0.2lf”这样指定读入数据格式的精度！只能写”%lf”这样
  • printf中”%.2f”这种精度规定会自动帮你四舍五入，不需要手动+0.5
  • switch语句别忘了加break
  • 区分++i和i++的区别！前者返回i+1后者返回i！
  • 最好不要用变量定义数组大小！

2.4-错误点

第三章 函数抽象与递归跳跃

3.1-递归的根本是结束

3.1-错误代码

题目：输入一个数 $N (1\leq N \leq 10^3)$ ，输出 $f(N)$。满足 $\forall x(3\leq x \leq N), f(x)=f([x/2])+f(x-1)$，其中方括号代表取整，而 $f(1)=1,f(2)=3$ 。

3.1-错误点

思路是对的，就是去在函数中重复调用该函数去计算之前的x的值，但是需要注意，如果只这么写的话，函数就会无限自我调用而不会停止。一定要记住——写递归函数的根本是结束条件。如果说递归函数的基本构成元素是“调用自身”，那么使它有效而不可或缺的元素就是“结束条件”，由于递归比较难以理解，第三章的问题都会后附正确代码以供理解，当然也推荐自己先修改完原代码再看正确代码。

注：本题代码有两个错误。

3.3-错误点

首先这份代码会TLE，我们简单分析一下原因：比如说计算 M(15) 的时候，函数中调用了 M(14),M(13),M(12) ，进一步下去的话，这三个函数又分别调用了 M(13),M(12),M(11),M(12),M(11),M(10),M(10),M(9),M(8)… ，发现了吗？有些值被重复调用了好几次，这些重复调用又会导致更多的重复调用，让程序消耗时间几何级别增长，最终就TLE了。这里我们采用一种“记忆化”的方法进行处理，就是用一个数组存储算过的值，如果这个值算过，就不再算，直接取出数组的值即可。

其次是，int最大值是 $2\times 10^9$ 左右，题目中这么计算的话，两次取模完加起来的值就已经可能接近int最大值了，三个加起来更是很容易爆int。所以说对于要求对答案取模的题目，一定要尽量多取模，能取模几次就取模几次。如果少取模一次可能会WA，但是多取模一定没影响。
